# Día 2: Modelado de Datos y Gestión de Memoria 🧱

## 🎯 Contexto Inicial
Java es un lenguaje de tipado fuerte. Esto significa que cada pieza de información tiene una naturaleza clara y definida. Hoy aprenderemos a seleccionar las estructuras adecuadas para cada escenario y entenderemos cómo se gestiona la memoria de estos elementos.

## 📚 Conceptos de Base
1.  **Tipos Primitivos (8)**: Representan valores simples almacenados directamente en el **Stack**. Es la forma más eficiente de manejar números y caracteres.
2.  **Literales y Sufijos**: Son marcas para ayudar al compilador a distinguir entre tipos de similar naturaleza (como `float` y `double`, o `int` y `long`).
3.  **Modern Java (Records)**: Estructuras diseñadas para el transporte inmutable de datos, eliminando la necesidad de código repetitivo.

## 🛠️ Reto Práctico: Experimento de Almacenamiento
1.  **Tipado Preciso**: Crea una clase `Primitivos.java`. Declara una variable para cada uno de los 8 tipos primitivos e inicialízala con un valor válido.
2.  **Análisis de Desbordamiento**: Declara un `byte n = 127;`. Luego realiza la operación `n = (byte)(n + 1);`. 
    *   *¿Qué valor imprime y por qué ocurre este comportamiento en un lenguaje de tipado estático?*
3.  **Representación Visual**: Utiliza un **Text Block** (`"""`) para mostrar un reporte del sistema `Corporate Talent Hub`.
4.  **Inmutabilidad Moderno**: Crea un `record Empresa(String nombre, int sedes)`. Intenta modificar una de sus propiedades una vez instanciado.
