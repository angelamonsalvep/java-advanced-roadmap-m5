# Día 1: Arquitectura de la Plataforma Java 🏗️

## 🎯 Contexto Inicial
Java es una de las plataformas de desarrollo más robustas y versátiles. Su potencia reside en la separación entre el desarrollo humano y la ejecución en hardware. Hoy exploraremos el ecosistema Java y su flujo de trabajo fundamental.

## 📚 Conceptos de Base
1.  **JDK (Java Development Kit)**: Es el entorno de desarrollo que contiene el compilador (`javac`). Su función es transformar el código fuente en un lenguaje intermedio.
2.  **Bytecode**: Es un formato de instrucciones de bajo nivel contenido en archivos `.class`. Es portable y constituye el lenguaje nativo de la JVM.
3.  **JVM (Java Virtual Machine)**: Es la máquina virtual responsable de ejecutar el Bytecode en el hardware del sistema operativo anfitrión.

---

## 🛠️ Reto Práctico: Compilación y Ejecución Manual
*El objetivo es realizar el ciclo de vida del software sin automatización.*

1.  Crea un archivo de texto llamado `Arquitectura.java`.
2.  Implementa un programa que muestre por consola el mensaje: `"Compilación con JDK y ejecución con JVM"`.
3.  Utiliza la terminal para generar el Bytecode mediante: `javac Arquitectura.java`.
4.  **Análisis de Persistencia**: Elimina el archivo `.java` y ejecuta el programa usando `java Arquitectura`.
    *   *¿Qué componente es el responsable de que el programa siga funcionando tras eliminar el código fuente?*
5.  **Desensamblado**: Ejecuta el comando `javap -c Arquitectura.class` para observar las instrucciones internas del Bytecode.
