# Día 4: Estándares Industriales y Gestión de Memoria ⚙️

## 🎯 Contexto Inicial
Hoy daremos el paso de la gestión manual a la automatización industrial. Aprenderás a utilizar **Maven**, la herramienta que orquesta el ciclo de vida del software, y profundizaremos en cómo Java gestiona el almacenamiento físico en la memoria RAM.

## 📚 Conceptos de Base
1.  **Maven**: Es un gestor de proyectos que automatiza la compilación, la estructura de carpetas y las dependencias según estándares internacionales.
2.  **Stack vs Heap**: Son las dos áreas principales de memoria en la JVM. 
    - **Stack**: Memoria pequeña y veloz para variables locales y ejecución de hilos.
    - **Heap**: Área principal donde residen los objetos instanciados.
3.  **Garbage Collector (GC)**: El proceso automático que limpia la memoria de objetos que ya no son referenciados, optimizando el rendimiento del sistema.

## 🛠️ Reto Práctico: Adopción Industrial
1.  **Validación de Herramientas**: Verifica la instalación de Maven con el comando `mvn -version`.
2.  **Inicialización de Proyecto**: Ejecuta `mvn archetype:generate`. Selecciona la opción `quickstart`. 
    *   *Analiza la estructura de carpetas generada.*
3.  **Prueba de Diagnóstico**: En la clase `Main` generada por Maven, asigna `null` a una variable de tipo objeto e intenta llamar a una de sus propiedades. 
    *   *Analiza el mensaje de error que proporciona Java 17+.*
4.  **Limpieza del Proyecto**: Ejecuta `mvn clean compile` y observa la creación del directorio `target/`.
