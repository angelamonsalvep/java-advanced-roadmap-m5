# Día 1: Conexión Segura - JDBC Moderno 💾

## 🎯 Contexto Inicial
Hoy aprenderás a conectar tu aplicación con el mundo exterior. En el pasado (Legacy), gestionar conexiones a bases de datos era un proceso peligroso que causaba fugas de memoria (Memory Leaks). Hoy dominarás el **JDBC moderno** y el cierre automático de recursos con Java 17+.

## 📚 Conceptos de Base
1.  **JDBC (Java Database Connectivity)**: El estándar oficial para conectar Java con bases de datos relacionales.
2.  **Try-with-Resources**: Una estructura moderna que asegura que la conexión, el statement y el resultset se cierren solos, incluso si ocurre un error.
3.  **Memory Leaks**: Ocurren cuando dejas conexiones abiertas. El sistema se ralentiza hasta que se bloquea por falta de recursos.

## 🛠️ Reto Práctico: El Túnel de Datos
1.  **Utilidad de Conexión**: Crea una clase de utilidad para la conexión JDBC hacia una base de datos relacional.
2.  **Análisis Histórico**: Investiga y escribe en un comentario cómo se cerraban las conexiones antes de Java 7 usando el bloque `finally`.
3.  **Implementación Robusta**: Implementa la conexión obligatoriamente usando el bloque **`try-with-resources`** de Java 17+.
    *   *Analiza por qué este código es más breve y seguro frente a errores de ejecución.*
