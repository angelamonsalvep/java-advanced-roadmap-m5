El Estándar JDBC

JDBC (Java Database Connectivity) no es solo una librería; es una capa de abstracción que permite a Java hablar con cualquier base de datos relacional.

  • Independencia del Motor: El mismo código Java puede funcionar en MySQL, PostgreSQL u Oracle (cambiando solo el Driver).
  • Drivers: Son implementaciones específicas (JARs) que actúan como "traductores" entre las llamadas de Java y el protocolo nativo de la DB.
  • Conectividad: Maneja desde la autenticación hasta el envío de bytes sobre la red.
// Estructura de una URL JDBC
jdbc:postgresql://localhost:5432/mi_db

// Dependencia Maven (Ejemplo)
<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <version>42.7.2</version>
</dependency>

Adiós al Bloque Finally

Legacy (Pre Java 7)

Connection c = null;
try {
  c = ...
} catch(Exception e) { ... }
finally {
  if (c != null) try { c.close(); }
  catch (Exception e) { ... }
}

⚠️ Propenso a Memory Leaks. Si el cierre fallaba, la conexión quedaba "huérfana".

Moderno (Java 17+)

try (Connection c = getConn()) {
  // Operaciones aquí
} catch (SQLException e) {
  // Manejo de errores
}
// ¡Cierre garantizado aquí!

✅ Try-with-Resources: Implementa la interfaz AutoCloseable. Más limpio y 100% seguro.

Patrón DAO: Separación de Responsabilidades

Definición: El Data Access Object (DAO) abstrae y encapsula todo el acceso a la fuente de datos.
  • Desacoplamiento: La lógica de negocio no sabe si usas JDBC, un archivo o un API externa.
  • Mantenibilidad: Si la base de datos cambia su estructura, solo modificas la clase DAO, no toda la App.
  • Testabilidad: Permite usar "Mocks" para probar la lógica de negocio sin tener una base de datos real encendida.
// El Contrato (Interface)
public interface UserDAO {
  User findById(Long id);
  void save(User u);
}

// La Implementación
public class UserDAOImpl implements UserDAO { ... }

Seguridad: Inyección SQL

El Peligro: Concatenación

String sql = "SELECT * FROM users WHERE pass = '" + input + "'";

Si el usuario escribe: ' OR '1'='1

El query se convierte en:
SELECT * FROM users WHERE pass = '' OR '1'='1'

Resultado: El atacante entra sin contraseña.

La Solución: PreparedStatement

String sql = "SELECT * FROM users WHERE pass = ?";
PreparedStatement ps = conn.prepareStatement(sql);
ps.setString(1, input);
  • Pre-compilación: La DB ya sabe qué es código y qué es dato.
  • Escapado Automático: El input se trata como texto literal, nunca como código ejecutable.

Connection Pooling: El Estándar Pro

Abrir una conexión física a la DB es COSTOSO (negociación TCP, autenticación, handshake SSL).

  • ¿Cómo funciona? La aplicación mantiene un "pool" de conexiones ya abiertas y listas para usar.
  • Ciclo: Pides una -> La usas -> La "devuelves" al pool (no se cierra físicamente).
  • Herramientas: HikariCP es el estándar de facto en Java por su increíble velocidad.
💡 Dato curioso: Abrir una conexión nueva puede tardar 50ms-100ms. Tomar una del pool tarda <1ms.

Mapping: Del Mundo Relacional al Mundo de Objetos

Las bases de datos devuelven tablas; Java usa objetos. El DAO debe realizar esta traducción.

try (ResultSet rs = ps.executeQuery()) {
  if (rs.next()) {
    return new User(
      rs.getLong("id"),
      rs.getString("username"),
      rs.getTimestamp("created_at").toLocalDateTime()
    );
  }
}

Este proceso manual es lo que automatizan los ORMs como Hibernate o JPA en etapas posteriores.

Checklist del Desarrollador Java

  • 🚀 Eficiencia: Usa Connection Pools.
  • 🛡️ Seguridad: NUNCA concatenes Strings en SQL.
  • 🧹 Limpieza: Siempre usa Try-with-resources.
  • 🏗️ Arquitectura: Separa tus DAOs del resto del código.
  • ⚖️ Consistencia: Aprende ACID y gestiona transacciones (commit/rollback).
