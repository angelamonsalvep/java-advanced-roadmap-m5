# Día 3: Organización Profesional - Arquitectura MVC 🏗️

## 🎯 Contexto Inicial
Hoy aprenderás a organizar tu código como un profesional. En aplicaciones reales, no puedes tener la captura de datos (Scanner) mezclada con el SQL. Descubrirás el **Patrón MVC** (Modelo-Vista-Controlador) para que cada clase tenga una sola responsabilidad clara.

## 📚 Conceptos de Base
1.  **Model**: Donde reside el estado (Entidades) y el acceso a datos (DAO).
2.  **View**: Donde reside la interacción con el usuario (el `Scanner` y los `println`).
3.  **Controller**: El cerebro que coordina. Recibe datos de la Vista y llama a los métodos del Modelo.

## 🛠️ Reto Práctico: Refactorización a Escala Industrial
1.  **División de Paquetes**: Organiza tu proyecto en paquetes: `com.riwi.talent.model`, `com.riwi.talent.controller` y `com.riwi.talent.view`.
2.  **Vista Pura**: Asegúrate de que toda la interacción con el usuario (`Scanner`) ocurra **únicamente** en la capa de Vista.
3.  **El Controlador**: Implementa el Controlador como mediador que recibe la petición de la vista y coordina las acciones con el modelo de datos.
