Día 3: Organización Profesional

1. El Patrón Modelo-Vista-Controlador (MVC)

El patrón MVC es un estándar de arquitectura que separa los datos, la interfaz de usuario y la lógica de control para lograr un sistema mantenible y escalable.

Modelo (Model)

Gestiona los datos y la lógica de negocio. Incluye las entidades (POJOs/Records) y el acceso a datos (DAO). No conoce la interfaz de usuario.

Vista (View)

Es la interfaz con la que interactúa el usuario. En aplicaciones de consola, es donde residen únicamente los Scanner y System.out.

Controlador (Controller)

Actúa como cerebro coordinador. Recibe las peticiones de la Vista, solicita datos al Modelo y decide qué mostrar en la Vista.

2. Separación de Responsabilidades (SoC)

Organizar el proyecto en paquetes evita el "espagueti de código" y facilita el mantenimiento por parte de múltiples desarrolladores.

// Estructura de Paquetes Profesional
com.riwi.talent
├── controller   // Recibe inputs y coordina flujos
├── model        // Lógica de persistencia y entidades
│   ├── dao      // Interfaces y clases JDBC (CRUD)
│   └── entity   // Records para inmutabilidad
└── view         // Interfaz de consola pura
Día 4: Integración Avanzada

3. Integración: Records + JDBC

Los Records (Java 17+) son la herramienta perfecta para el transporte de datos inmutables entre capas JDBC y la Vista.

Ejemplo de Implementación DAO:

// Definición del Record (Inmutabilidad total)
public record Curso(int id, String nombre, String duracion) {}

// Uso en la capa de datos
public List<Curso> obtenerCursos() {
    List<Curso> cursos = new ArrayList<>();
    try (Connection conn = DatabaseConnection.getConnection();
         Statement stmt = conn.createStatement();
         ResultSet rs = stmt.executeQuery("SELECT * FROM cursos")) {
        
        while (rs.next()) {
            cursos.add(new Curso(
                rs.getInt("id"),
                rs.getString("nombre"),
                rs.getString("duracion")
            ));
        }
    } catch (SQLException e) { e.printStackTrace(); }
    return cursos;
}

4. Reportes con Text Blocks

Los Text Blocks permiten definir plantillas de reporte multilínea que son fáciles de leer y mantener, evitando la concatenación con el operador +.

public void imprimirReporte(List<Curso> cursos) {
    String reporteTemplate = """
        ********************************************
        *       REPORTE CORPORATIVO DE CURSOS      *
        ********************************************
        * %-5s | %-20s | %-10s *
        --------------------------------------------
        %s
        ********************************************
        """;
    
    StringBuilder filas = new StringBuilder();
    cursos.forEach(c -> filas.append(String.format("* %-5d | %-20s | %-10s *\n", 
        c.id(), c.nombre(), c.duracion())));

    System.out.printf(reporteTemplate, "ID", "NOMBRE DEL CURSO", "DURACIÓN", filas);
}
