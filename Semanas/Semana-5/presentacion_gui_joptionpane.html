Interfaces Gráficas con JOptionPane

Evolución del Patrón MVC hacia entornos visuales en Java

Concepto

1. De la Consola al Diálogo Gráfico

En el desarrollo corporativo, la interacción mediante Scanner es limitada. JOptionPane de la librería javax.swing permite crear cuadros de diálogo modales para entrada y salida de datos sin necesidad de diseñar ventanas complejas.

Vista de Consola (Scanner)
- Texto plano secuencial.
- Difícil validación visual.
- Experiencia de usuario rústica.
Vista GUI (JOptionPane)
- Ventanas modales emergentes.
- Iconos de estado (Error, Info).
- Interacción más intuitiva.
Implementación

2. Captura de Datos Inmutable (Records)

Utilizamos showInputDialog para recibir datos del usuario y los encapsulamos inmediatamente en un Record para enviarlos al Controlador.

// Captura de datos en la Vista
String nombre = JOptionPane.showInputDialog("Ingrese el nombre del curso:");
String duracion = JOptionPane.showInputDialog("Ingrese la duración:");

// Empaquetado en Record para el Modelo
Curso nuevoCurso = new Curso(0, nombre, duracion);
Visualización

3. Reportes con Text Blocks en GUI

Podemos combinar la potencia de los Text Blocks de Java 17 con showMessageDialog para presentar tablas de datos formateadas con fuentes monoespaciadas.

public void mostrarCursos(List<Curso> cursos) {
    StringBuilder sb = new StringBuilder();
    cursos.forEach(c -> sb.append(String.format("ID: %d | %-20s | %s\n", 
        c.id(), c.nombre(), c.duracion())));

    String reporte = """
        LISTADO DE CURSOS ACTIVOS
        -----------------------------------
        """ + sb.toString();

    JOptionPane.showMessageDialog(null, reporte, 
        "Sistema de Gestión", JOptionPane.INFORMATION_MESSAGE);
}
Arquitectura

4. El Flujo de Control en la GUI

El Controlador sigue siendo el cerebro. No importa si los datos vienen de una consola o de un diálogo Swing; la lógica de negocio y el acceso a datos (JDBC) permanecen intactos.

// En el Controller
public void ejecutar() {
    int opcion = Integer.parseInt(JOptionPane.showInputDialog("1. Crear\n2. Listar\n3. Salir"));
    
    switch(opcion) {
        case 1 -> {
            Curso c = vista.pedirDatos(); // Usa JOptionPane
            modelo.guardar(c);            // Usa JDBC
        }
        case 2 -> {
            vista.mostrarCursos(modelo.listar()); 
        }
    }
}

Ventajas de la Integración

  • Desacoplamiento: Cambiar de Scanner a JOptionPane solo afecta a la capa de la Vista.
  • Validación: Facilidad para manejar entradas nulas o cancelaciones mediante las respuestas de los diálogos.
  • Estándar Corporativo: Es el primer paso antes de migrar a interfaces más complejas como JavaFX o Aplicaciones Web.
