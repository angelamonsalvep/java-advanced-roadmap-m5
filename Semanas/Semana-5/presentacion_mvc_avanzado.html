Dominando el Patrón MVC

Sinergia entre POO, Capas de Datos y Arquitectura Profesional

1. Anatomía Profunda de las Capas

Modelo (The Source)

No es solo una clase; es un ecosistema. Contiene las Entidades (estado), DAOs (persistencia) y la lógica que garantiza la integridad de los datos.

Vista (The Face)

Totalmente pasiva. Su única misión es capturar eventos del usuario y presentar datos. Regla de Oro: Nunca debe instanciar un DAO ni Connection.

Controlador (The Brain)

Orquestador de flujos. Implementa la lógica de navegación y coordina la comunicación entre la Vista y el Modelo, actuando como un túnel de datos.

2. MVC y los Pilares de la POO

El MVC no es ajeno a la POO; es su aplicación práctica más potente:

  • Encapsulamiento: Cada capa oculta su complejidad. La Vista no sabe que existe SQL; el Modelo no sabe que existe un botón.
  • Abstracción: Usamos interfaces (ej: ICursoDAO) para definir el "qué" y clases de implementación para el "cómo".
  • Polimorfismo: Podemos cambiar la implementación de un DAO (de MySQL a Oracle) sin tocar una sola línea de la Vista o el Controlador.
// Abstracción mediante Interfaz
public interface ICrudDAO<T> {
    void save(T entity);
    List<T> findAll();
}
Buenas Prácticas

3. Externalización: config.properties

Nunca debemos dejar credenciales (User, Pass, URL) "hardcodeadas" en el código. Usar un archivo .properties permite cambiar el entorno sin recompilar la aplicación.

Archivo db.properties:

db.url=jdbc:mysql://localhost:3306/mi_base_datos
db.user=root
db.password=secreto123

Carga en Java (Clase DatabaseConnection):

Properties props = new Properties();
try (FileInputStream fis = new FileInputStream("db.properties")) {
    props.load(fis);
    String url = props.getProperty("db.url");
    // Crear conexión...
}

4. El Ciclo de Vida de una Petición

1. VISTA: Captura input (ej: JOptionPane.showInputDialog).
2. VISTA: Llama al método del CONTROLADOR pasando el objeto.
3. CONTROLADOR: Valida los datos básicos.
4. CONTROLADOR: Invoca al MODELO (DAO) para persistencia.
5. MODELO: Ejecuta JDBC y retorna confirmación.
6. CONTROLADOR: Recibe respuesta y ordena a la VISTA mostrar éxito/error.

Conclusión Arquitectónica

El MVC profesional no solo divide archivos; divide **responsabilidades mentales**. Al combinarlo con POO sólida y configuración externa, creamos sistemas que están listos para el mundo real: seguros, testeables y fáciles de escalar.
