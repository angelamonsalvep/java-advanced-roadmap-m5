# Día 1: Evolución del Control de Flujo - Switch Moderno 🔄

## 🎯 Contexto Inicial
El control de flujo es la columna vertebral de cualquier algoritmo. Durante años, el `switch` clásico fue el estándar, pero traía consigo riesgos de legibilidad y errores por omisión (fall-through). Hoy compararemos cómo Java ha evolucionado hacia las **Switch Expressions**.

## 📚 Conceptos de Base
1.  **Switch Clásico (Java 8)**: Basado en etiquetas `case` y la palabra clave `break`. Olvidar un `break` provoca que el código siga ejecutando los casos siguientes por error.
2.  **Switch Expression (Java 17/21)**: Utiliza la sintaxis de flecha `->`. Es exhaustiva, más breve y, lo más importante, puede devolver un valor (actúa como una expresión).
3.  **Yield**: Una palabra clave especial para devolver valores en bloques de Switch complejos.
