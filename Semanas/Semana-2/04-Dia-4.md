# Día 4: Robustez y Diagnóstico de Errores 🛡️

## 🎯 Contexto Inicial
Un buen programador no solo escribe código que funciona, sino código que no se rompe cuando las cosas fallan. Hoy aprenderás a capturar errores inesperados y a utilizar el operador ternario para simplificar la toma de decisiones.

## 📚 Conceptos de Base
1.  **Excepciones (`try-catch`)**: Permiten capturar un error (ej: el usuario escribe texto donde se pide un número) y manejarlo sin que el programa se cierre.
2.  **InputMismatchException**: Se lanza cuando el tipo de dato recibido por el `Scanner` no coincide con el esperado.
3.  **Operador Ternario**: Una forma abreviada de `if/else` ideal para asignaciones rápidas (ej: `String estado = (nota > 60) ? "Aprobado" : "Reprobado";`).

---

## 🛠️ Reto Práctico: Un Sistema Inquebrantable
1.  **Captura Segura**: Envuelve la captura de datos del `Scanner` en un bloque `try-catch`. Si el usuario introduce una letra en lugar de un ID numérico, el sistema debe informar el error y continuar.
2.  **Estado de Promoción**: Utiliza un **Operador Ternario** para determinar si un empleado es elegible para una promoción basada en su promedio de la matriz de ayer.
3.  **Diagnóstico Moderno**: Analiza los mensajes de error de tus excepciones.
