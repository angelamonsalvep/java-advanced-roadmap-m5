# Día 1: El Fin de los Arreglos Fijos - ArrayList 🚀

## 🎯 Contexto Inicial
En el mundo real, nunca sabes cuántos empleados tendrá tu empresa. Usar un arreglo de tamaño fijo (como `Empleado[10]`) es peligroso e ineficiente. Hoy aprenderás a usar el **Java Collections Framework**, la herramienta que permite que tus estructuras de datos crezcan y se encojan según la necesidad.

## 📚 Conceptos de Base
1.  **Colecciones vs Arreglos**: Un arreglo tiene un tamaño inmutable. Una colección es dinámica.
2.  **ArrayList<T>**: La lista más común de Java. Está respaldada por un arreglo que crece automáticamente. 
3.  **Genéricos (<>)**: El mecanismo de Java para decir: "En esta lista *solo* guardaremos objetos de tipo Empleado". Esto evita errores en tiempo de ejecución.

## 🛠️ Reto Práctico: Migración de Datos
1.  **Refactorización Core**: En tu proyecto industrial, sustituye los arreglos fijos de la Semana 2 por un `ArrayList<Empleado>`.
2.  **Operaciones Dinámicas**: Implementa los métodos para **Agregar**, **Eliminar** por índice y **Listar** todos los coders.
3.  **Observación Técnica**: Imprime el tamaño de la lista (`.size()`) antes y después de cada operación para ver cómo cambia dinámicamente.

## 🔍 Temas de Investigación (Más allá de la IA)
1.  **Internals de ArrayList**: Investiga cómo hace Java para "crecer" el arreglo interno cuando se llena. ¿Qué es el "Capacity" y el "Load Factor"?
2.  **Complejidad Algorítmica (Big O)**: Investiga por qué agregar un elemento al final de un `ArrayList` es rápido (O(1)), pero eliminar el primer elemento es lento (O(n)).
3.  **Documentación Oficial**: Busca en los Oracle Docs la diferencia entre `add(E e)` y `add(int index, E element)`. ¿Qué pasa con los otros elementos cuando insertas uno en medio?
