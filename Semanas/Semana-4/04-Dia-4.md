# Día 4: Evolución de Contratos - Interfaces y Métodos Default 🧬

## 🎯 Contexto Inicial
Hoy aprenderás a diseñar contratos que puedan evolucionar sin romper el código existente. Antes de Java 8, añadir un método a una interfaz obligaba a modificar todas las clases que la implementaban. Hoy dominarás los **Default Methods**, una de las innovaciones que permitió la evolución de las librerías modernas.

## 📚 Conceptos de Base
1.  **Interface**: Un contrato de comportamiento. Define *qué* debe hacer un objeto, pero no *cómo*.
2.  **Default Methods**: Permiten añadir funcionalidad a una interfaz sin romper las clases existentes. Son la base de las colecciones de Java 8+.
3.  **Encapsulamiento Avanzado**: Proteger el estado interno de un objeto mediante modificadores de acceso (`private`, `protected`).

## 🛠️ Reto Práctico: El Contrato de Promoción
1.  **Diseño de Contrato**: Crea la interfaz `Promocionable` con un método abstracto para calcular bonos de ascenso.
2.  **Evolución Segura**: Implementa un **método `default`** en la interfaz que registre un log automático cada vez que se calcula un bono de promoción.
3.  **Encapsulamiento**: Asegúrate de que todos los atributos de tu jerarquía (`Persona`, `Empleado`, `Desarrollador`) utilicen correctamente los modificadores `private` y `protected`.
