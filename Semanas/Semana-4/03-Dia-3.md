# Día 3: Código Limpio - Polimorfismo y Pattern Matching 🧹

## 🎯 Contexto Inicial
Hoy aprenderás a simplificar la lógica de tu sistema eliminando el código repetitivo de conversión. Durante décadas (Legacy), el uso de `instanceof` obligaba a realizar un casting manual que ensuciaba el código. Hoy dominarás la forma moderna de gestionar el polimorfismo con Java 17+.

## 📚 Conceptos de Base
1.  **Pattern Matching for `instanceof`**: Permite preguntar por el tipo de un objeto y, si coincide, crear una variable tipada automáticamente en una sola línea.
2.  **Polimorfismo**: La capacidad de un objeto de tomar múltiples formas. Una `Persona` puede actuar como `Desarrollador` o `Gerente`.
3.  **Casting Manual**: La técnica antigua que hoy consideramos "boilerplate" innecesario y propenso a errores.

---

## 🛠️ Reto Práctico: Roles Específicos sin Casting
1.  **Especialización**: Crea las subclases `Desarrollador` (con atributo `lenguajePrincipal`) y `Gerente` (con atributo `presupuestoMensual`).
2.  **Lógica Dinámica**: Implementa un método de validación que use **Pattern Matching** (ej: `if (p instanceof Desarrollador des)`) para acceder directamente a los métodos específicos de cada rol sin hacer casting manual.
3.  **Comparación Técnica**: Escribe en un comentario cómo habrías hecho esto en Java 8 y por qué el Pattern Matching de Java 17 es más seguro ante errores de tipo.
