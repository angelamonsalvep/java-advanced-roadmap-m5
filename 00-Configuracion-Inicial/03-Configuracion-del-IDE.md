# Guía Maestra de Entornos de Desarrollo (IDEs) 🖥️

Un IDE no es solo un editor de texto; es un ecosistema que integra compilación, depuración (debugging), gestión de dependencias y refactorización. Para esta ruta avanzada, es fundamental que domines al menos uno de los estándares de la industria.

## 🏆 Los "Cuatro Grandes" de Java
Haz clic en el IDE de tu elección para ver su guía detallada de instalación y configuración:

1.  **[IntelliJ IDEA](IDE-INTELLIJ.md)**: El estándar de oro moderno. Preferido por su inteligencia y productividad.
2.  **[Eclipse IDE](IDE-ECLIPSE.md)**: El veterano robusto. Esencial en sectores bancarios y grandes corporativos.
3.  **[Visual Studio Code (VSC)](IDE-VSC.md)**: El retador ligero. Ideal para microservices y entornos políglotas.
4.  **[Apache NetBeans](IDE-NETBEANS.md)**: El oficial de la comunidad. Excelente para el aprendizaje y aplicaciones GUI.

## 🔍 Investigación Técnica sobre IDEs
1.  **Indexación**: ¿Por qué un IDE consume tanta memoria RAM al abrir un proyecto por primera vez? Investiga el concepto de "Indexación de archivos".
2.  **LSP (Language Server Protocol)**: ¿Cómo logra Visual Studio Code entender Java si no fue diseñado originalmente para Java? Investiga el rol de Red Hat en el soporte de Java para VSC.
3.  **Refactorización**: ¿Cuál es la diferencia entre "Rename" y "Find and Replace"? ¿Por qué el IDE es más seguro para cambiar el nombre de una clase?
4.  **Gemini CLI**: `gemini "Explícame qué es el Debugger de un IDE y cómo funcionan los Breakpoints para detener la ejecución del código"`
